# AegisOps Mobile — руководство

## Установка

1. Скачайте APK из [Releases](https://github.com/FreedoomForm/goal/releases):
   - `AegisOps-Mobile-universal.apk` — совместим со всеми архитектурами (~40 MB)
   - `AegisOps-Mobile-arm64.apk` — только 64-бит ARM (меньше размер)
2. На Android: **Настройки → Безопасность → Разрешить установку из неизвестных источников**.
3. Откройте скачанный APK и нажмите «Установить».

## Первое подключение

### Вариант A: QR-код (рекомендуется)

1. На ПК запустите AegisOps Desktop.
2. Перейдите в **Мобильный доступ** (левая панель).
3. Нажмите **«Запустить Cloudflare»** — это создаёт публичный HTTPS URL.
   - Альтернатива: установите свой ngrok auth token в `NGROK_AUTHTOKEN` и используйте его.
   - Альтернатива: введите URL вручную, если у вас свой reverse-proxy.
4. Нажмите **«Создать код сопряжения»** — появится 6-значный код и QR.
5. В APK нажмите **«Сканировать QR»**, наведите камеру — APK получит URL и API-key, сохранит в Android KeyStore и перейдёт на главный экран.

### Вариант B: Ручной ввод

Если камеры нет или QR не сканируется:
1. В APK разверните секцию **«...или введите вручную»**.
2. Введите URL сервера (например, `https://abc-def.trycloudflare.com`).
3. Введите 6-значный код.
4. Нажмите **«Подключиться»**.

### Вариант C: LAN (без интернета)

Если ПК и телефон в одной Wi-Fi сети:
1. На ПК запустите `BIND=0.0.0.0 npm run server` или `npm run server:lan`.
2. Узнайте IP ПК: `ipconfig` (Windows) / `ifconfig` (macOS/Linux). Например, `192.168.1.42`.
3. В десктоп-UI создайте API-key: **Мобильный доступ → Создать код сопряжения**.
4. В APK вручную введите URL: `http://192.168.1.42:18090` и код сопряжения.

Android разрешает cleartext HTTP только для LAN-диапазонов (192.168.x.x, 10.x.x.x, 172.16-31.x.x). Для публичного интернета HTTPS обязателен.

## Возможности мобильного клиента

| Вкладка | Что доступно |
|---|---|
| 🏠 Панель | Обзор: коннекторы, сценарии, модули, документы |
| ▶️ Сценарии | Список + запуск любого сценария одним тапом |
| 💬 AI | Чат с Ollama (через ПК), markdown-рендеринг ответов |
| 🔀 Workflow | Список сохранённых workflows + запуск + трассировка |
| 🧩 MCP | Управление MCP-серверами (start/stop/list tools) |
| ⚙️ Настройки | Отключиться, увидеть URL/ключ |

Все вкладки — **фасад над API ПК**. Мобильное приложение ничего не считает локально: все запросы проксируются в ПК, где работает Ollama, 1С-коннекторы, SCADA и т. д.

## Безопасность на устройстве

- API-ключ хранится в `flutter_secure_storage` → шифруется ключом из Android KeyStore (AES-256).
- При удалении приложения ключи удаляются.
- **Отзыв компрометированного ключа**: на ПК **Мобильный доступ → API-ключи → Отозвать**. APK сразу перестанет работать.
- Автоотзыв по времени: не реализован (планируется). Пока — вручную.

## Сборка APK из исходников

```bash
cd android_app
flutter pub get
flutter build apk --release --split-per-abi
# Результат: build/app/outputs/flutter-apk/
```

Требования: Flutter SDK 3.22+, Android SDK 34, JDK 17.

В CI это делается автоматически при каждом push в main (см. `.github/workflows/build-android.yml`).

## Известные ограничения

- **Нет визуального редактора Workflow** на мобильном (слишком сложно для touch-UI). Вместо этого — список + запуск. Редактировать нужно на ПК.
- **Нет push-уведомлений** (пока). Планируется через FCM + сервер → подписки на события сценариев.
- **Offline-режим отсутствует**. Приложение требует соединения с ПК-сервером.
- **iOS версии нет** — Flutter код совместим, но нет инфраструктуры для App Store публикации. При необходимости можно легко добавить `ios/` directory и `build-ios.yml` в CI (потребуется Apple Developer account).
